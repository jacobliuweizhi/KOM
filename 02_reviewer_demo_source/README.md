# KOM Reviewer Demo

`KOM Reviewer Demo` is a fully local, reviewer-facing demonstration of the **Knee Osteoarthritis Manager (KOM)**, a physician-side AI decision-support workflow for standardized knee osteoarthritis treatment-planning tasks.

The demo is not a patient self-testing tool. It shows how standardized KOA case inputs are organized into assessment support, evidence-routed treatment recommendations, safety audit, clinician simulation, and manuscript-facing results.

## Open Directly

For reviewer delivery, use the single-file build. It does not require a server, API key, internet connection, or manual URL entry.

Recommended files:

- `start.bat`: Windows one-click launcher. Double-click it and the browser opens automatically.
- `index.html`: double-clickable single-file version.
- `KOM_Reviewer_Demo_Single_File.html`: same content with an ASCII filename for maximum compatibility across computers.

## Page guide

- **Overview**: physician-facing KOM study framework, four evidence chains, and sample counts.
- **Case workspace**: choose one standardized KOA case and run the complete workflow: Case input -> KOM-Profile -> KOM-Rad -> KOM-Risk -> KOM-RAG -> KOM-MDT -> KOM-Rx -> KOM-Safe -> Export report.
- **KOM-Assess**: profile, radiographic support, and longitudinal risk support.
- **KOM-Treat**: KOM-KB evidence summary, KOM-RAG retrieval, naive RAG baseline comparison, KOM-MDT, KOM-Rx, and KOM-Safe.
- **KOM-Sim**: clinician-in-the-loop standardized prescription task.
- **KOM-Score**: multi-source scoring anchors, ICC, and error definitions.
- **Results**: component ablation, clinician simulation, RAG, imaging, assessment, and risk results.
- **Methods**: manuscript-style methods and metric definitions.
- **Glossary**: searchable definitions for all KOM terms and study conditions.

## Key terms

**naive RAG baseline**: single-stage vector top-k retrieval over the same evidence library and query, without guideline anchors, evidence hierarchy, specialty routing, safety labels, graph links, or evidence arbitration.

**safety-critical error**: a prescription problem that may cause material patient harm, violates a contraindication, inappropriately recommends or delays needed referral, misuses high-risk medication, or omits essential risk gating.

**physician-task prescription record**: one prescription generated by one physician in one standardized prescription task. In KOM-Sim, 26 physicians each completed 30 tasks, yielding 780 physician-task prescription records. It does not mean 780 independent physicians or 780 independent cases.

## Reviewer demonstration route

1. Open Overview and explain that KOM is a physician-side KOA treatment decision-support system.
2. Choose a Q4 complex case in Case workspace.
3. Run the full KOM workflow.
4. Show KOM-RAG versus the naive RAG baseline in KOM-Treat.
5. Show KOM-Safe medication, exercise, and referral boundary checks.
6. Open KOM-Sim and compare Clinician + KOM with Clinician + KOM-R.
7. Open Results and review Full KOM, component ablation, KOM-Sim, and KOM-RAG metrics.
8. Export the markdown case report.

## Data note

All data are embedded local standardized demonstration cases and manuscript summary metrics. No backend, API key, external dataset, or network request is required after dependencies are installed.
